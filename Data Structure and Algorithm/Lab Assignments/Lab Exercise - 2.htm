Lab 2 – Binary Search

Use the source code for orderedArray.java program given in the book by Robert Lafore (Listing 2.4), and do the following exercises. No submission is required for this lab. However, you must get your work checked by a lab TA.

1.  The find() method in the orderedArray.java program uses the binary search algorithm. Test the method as follows.

a.          Verify that it can correctly find the first entry.

b.          Verify that it can correctly find the last entry.

c.          Verify that it terminates and returns nElems for something that would fall in the middle of the array and is not there.

d.          Verify that it terminates and returns nElems for something that comes before the first entry.

e.          Verify that it terminates and returns nElems for something that comes after the last entry.

f.            What does it do if there are duplicate keys?

2. Design a variant of the binary search algorithm that performs only one binary comparison of searchKey with an array entry, per function invocation. You may add additional comparisons on range variables. Test the correctness of your algorithm.

3.  Write a program to count the number of times the search loop of the binary search algorithm executes when searching for an integer contained in a one-dimensional array of sorted integers in the range 1 to 65,000. Use the Java method random in the Math class to generate the integer, targetInt, to be located (i.e., targetInt = 1 + (int) Math.random()*65,000). The program should accept an input, n, the number of integers to be located, and then output the average number of times the loop executed after locating the n randomly generated target integers. For comparative purposes, also output the log265,000.
