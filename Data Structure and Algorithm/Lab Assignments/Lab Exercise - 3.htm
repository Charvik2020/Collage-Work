Lab 3 – Stacks and Queues

You can refer to the implementations of the stack and queue given in the form of Java classes (myStack

, Queue).

 

  1. We can implement a stack using a queue. Write and test a class named StackX that is based on the Queue class. The StackX class should have the same methods and capabilities as the myStack class. The implementation of the methods should be achieved using only the methods provided by the Queue class. Hint: To push an element, simply enqueue it. To pop an element, dequeue all the elements and enqueue them, except the last one which you should return. The disadvantage of this stack implementation is that pop runs in O(n) time. 

 

  1. We can implement a queue using two stacks. Write and test a class named QueueX that is based on the myStack class. The QueueX class should have the same methods and capabilities as the Queue class. The implementation of the methods should be achieved using only the methods provided by the myStack class. Hint: To enqueue an element, simply push it onto one stack. To dequeue an element, pop all elements from the previous stack onto another stack, perform a pop operation on the second stack, return the value returned by the pop operation. The disadvantage of this queue implementation is that dequeue runs in O(n) time.

 

  1. We can use a linked list instead of an array to store the stack’s elements. Write and test a class named LinkedStack that is based on the LinkList class in Lab 2. The LinkedStack class should have the same methods (except the isFull method) and capabilities as the myStack class. The implementation of these methods should be achieved using only the methods provided by the LinkList class.

 

Add and test the following conversion method for the LinkedStack class; it returns the equivalent myStack:

 

myStack toArrayStack()

 

  1. We can use a linked list instead of an array to store the queue’s elements. Write and test a class named LinkedQueue that is based on the LinkList class developed by you in one of the Lab Assignments. The LinkedQueue class should have the same methods (except the isFull method) and capabilities as the Queue class. The implementations of these methods should be achieved using only the methods provided by the LinkList class.

 

Add and test the conversion method:

 

Queue toArrayQueue()

 

for the LinkedQueue class. The method returns the equivalent Queue.
