Lab 4 - Binary Trees

1.      You can refer to the Java program - tree.java from the book by Robert Lafore. The Tree class implements a binary search tree. The TreeApp class features a primitive user interface which allows the user to choose an operation (finding, inserting, deleting, traversing, and displaying the tree). The display routine uses character output to generate a picture of the tree. The available commands are the characters s, i, f, d, and t, for show, insert, find, delete and traverse. The i, f, and d options ask for the key value of the node to be operated on. The t option gives you a choice of traversals: 1 for preorder, 2 for inorder, and 3 for postorder. The display shows the key values arranged in something of a tree shape; however, you’ll need to imagine the edges. Two dashes (--) represent a node that doesn’t exist at a particular position in the tree. The program initially creates some nodes so the user will have something to see before any insertions are made. You can modify the initialization code to start with any nodes you want, or with no nodes (which is good nodes).

a.       Extend the Tree class to include a method min that returns a reference to the node with the “smallest” key value in the tree.

b.      Extend the Tree  class to include a method max that returns a reference to the node with the “largest” key value in the tree.

c.       Extend the Tree  class to include a method height that returns the height of the tree.

d.      Extend the Tree  class to include a method isFull that determines if the tree is full or not.

Extend the user interface to allow the user to choose the following additional operations: finding minimum, finding maximum, finding height, finding whether tree is full or not.

2.      Write a Java program that transforms a postfix expression into a tree such as that shown in Figure 8.11 of Chapter 8. The approach is analogous to that of evaluating a postfix expression discussed in class. However, instead of storing operands on the stack, we store entire subtrees. We read along the postfix string.  Here are the steps when we encounter an operand.

a.         Make a tree with one node that holds the operand.

b.         Push this tree onto the stack.

Here are the steps when we encounter an operator:

a.         Pop two operand tree B and C off the stack.

b.         Create a new tree A with the operator in its root.

c.         Attach B as the right child of A.

d.         Attach C as the left child of A.

e.         Push the resulting tree back on the stack.

When you complete evaluating the postfix string, you should pop the one remaining item off the stack. Somewhat amazingly, this item is a complete tree depicting the algebraic expression. You can see the prefix and infix representations of the original postfix (and recover the postfix expression) by traversing the tree as we described. You’ll need to modify the Tree class from the tree.java program and the ParsePost class from the postfix.java program in Chapter 4 posted here (postfix. java).
